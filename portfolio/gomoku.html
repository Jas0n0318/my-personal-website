Java 五子棋 AI 核心演算法

基於權重評估與攻守策略的智慧決策系統

Java 17 Object-Oriented Swing GUI Algorithm

專案概述

這是一個使用純 Java 開發的單機版五子棋遊戲。專案的核心在於實現一個能與玩家進行策略對抗的 AI。

不同於深度學習，此 AI 採用了經典的「啟發式評估」演算法。透過掃描棋盤上的每一個空位，計算其「攻擊價值」與「防守價值」，並根據當前的難度設定來決定最佳落子點。

  • 支援三種難度調整 (簡單/普通/困難)
  • 即時運算落子權重 (不需連網)
  • 具備圖形化介面 (GUI)

AI 決策流程 (Decision Making)

AI 的每一次落子，都是經過以下四個步驟的精密運算：

1. 全盤掃描
遍歷 15x15 的棋盤陣列，找出所有尚未落子的空位 (EMPTY)。
2. 攻守模擬
針對每個空位，分別假設「如果我下這裡(攻擊)」和「如果對手下這裡(防守)」能連成幾顆子。
Attack Score Defense Score
3. 權重總分
依據難度係數 (Defense Weight)，將攻守分數加權合併。困難模式會更重視防守。
4. 擇優落子
比較所有空位的總分，選出分數最高的位置 (Best Move) 進行落子，並加入微小隨機數避免死板。

關鍵程式碼實作

以下是 AI 評估系統的核心片段。evaluation() 函式負責計算單點在八個方向上的連線潛力。

public void aiMove(int[][] grid) { double bestScore = -100000; // 根據難度調整防守係數 double defenseWeight = 1.0; if (difficultyLevel == 2) defenseWeight = 2.5; // 困難模式：極度重視防守 // 遍歷棋盤 for (int y = 0; y < BOARD_SIZE; y++) { for (int x = 0; x < BOARD_SIZE; x++) { if (grid[y][x] == EMPTY) { // 1. 計算進攻分數 grid[y][x] = AI_COLOR; double attack = evaluation(grid, x, y); // 2. 計算防守分數 (模擬玩家下這) grid[y][x] = PLAYER_COLOR; double defense = evaluation(grid, x, y); // 3. 加權總分 (加入微量隨機數增加變化性) double total = attack * 1.1 + defense * defenseWeight; total += Math.random() * 5; // 4. 更新最佳解 if (total > bestScore) { bestScore = total; bestX = x; bestY = y; } } } } }

技術亮點： 透過 getScore 函式，我們給予不同連線數極大的分數差距（例如 4 連子給 1000 分，5 連子給 10000 分），確保 AI 會優先選擇「致勝」或「救命」的關鍵點。
